================================================================================
KEY FEATURES IMPLEMENTED
================================================================================

1. DELIVERABLE MANAGEMENT
   - Add deliverables with client, type, deadline, priority, description
   - Update status through full lifecycle (planned → in_progress → review → delivered → approved)
   - Track actual vs estimated hours
   - Associate multiple milestones with deliverables
   - Full status change history with timestamps

2. CLIENT MANAGEMENT
   - Store client information (name, contact, email, phone)
   - Track retainer details (amount, tier, contract dates)
   - Status tracking (active, paused, churned)
   - View all deliverables for a client

3. DEADLINE & WORKLOAD TRACKING
   - Automatic deadline calculation based on deliverable type SLAs
   - Natural language deadline input (today, tomorrow, next friday, etc.)
   - Overdue detection and alerting
   - Workload summary across all clients

4. SEARCH & FILTERING
   - Full-text search across titles, descriptions, notes
   - Filter by client, status, type, priority
   - List specific client deliverables
   - Show overdue items across all projects

5. REPORTING & EXPORT
   - Generate professional status update emails for clients
   - Export to CSV or JSON for analysis
   - Statistics on delivery performance (delivery rate, avg time to deliver, overdue %)
   - Workload breakdown by status, priority, and type

6. DATABASE
   - SQLite local storage at ~/.nex-deliverables/deliverables.db
   - Indexes on common query patterns for performance
   - Full referential integrity with foreign keys
   - No external services or dependencies required

================================================================================
COMMANDS IMPLEMENTED (12 Total)
================================================================================

1. ADD - Add new deliverable
   nex-deliverables add --client "Name" --title "Title" --type website --deadline 2026-05-01

2. CLIENT - Manage clients (3 subcommands)
   nex-deliverables client add --name "Acme" --email "contact@acme.com"
   nex-deliverables client show --name "Acme"
   nex-deliverables client list [--status active]

3. LIST - List deliverables with filters
   nex-deliverables list [--client "Name"] [--status in_progress] [--overdue]

4. SHOW - View deliverable details
   nex-deliverables show 42
   nex-deliverables show "Title Search"

5. STATUS - Update deliverable status
   nex-deliverables status 42 delivered [--message "Signed off"]

6. MARK - Quick mark by title (searches)
   nex-deliverables mark "Homepage redesign" delivered

7. OVERDUE - Show all overdue deliverables
   nex-deliverables overdue

8. SEARCH - Full-text search
   nex-deliverables search "website logo"

9. WORKLOAD - Summary across all clients
   nex-deliverables workload

10. EMAIL - Generate status email for client
    nex-deliverables email "Acme Corp"

11. EXPORT - Export to CSV or JSON
    nex-deliverables export --format csv --client "Acme" --output file.csv

12. STATS - Delivery metrics and statistics
    nex-deliverables stats

================================================================================
DATABASE SCHEMA
================================================================================

CLIENTS TABLE:
  id (PK), name (UNIQUE), contact_name, email, phone, retainer_amount,
  retainer_tier, contract_start, contract_end, status, notes, created_at

DELIVERABLES TABLE:
  id (PK), client_id (FK), title, type, description, status, priority,
  assigned_to, deadline, started_at, delivered_at, approved_at,
  estimated_hours, actual_hours, notes, tags, created_at, updated_at

MILESTONES TABLE:
  id (PK), deliverable_id (FK), title, completed, completed_at, created_at

STATUS_UPDATES TABLE:
  id (PK), deliverable_id (FK), old_status, new_status, message, created_at

INDEXES:
  - deliverables(client_id)
  - deliverables(status)
  - deliverables(deadline)
  - clients(status)
  - milestones(deliverable_id)

USAGE EXAMPLES
================================================================================

# Setup
bash setup.sh

# Add a client
nex-deliverables client add --name "Ribbens Airco" \
  --contact-name "John Ribbens" --email "john@ribbens.be"

# Add deliverables
nex-deliverables add --client "Ribbens Airco" \
  --title "Homepage redesign" --type website --deadline 2026-05-01 --priority high

nex-deliverables add --client "Ribbens Airco" \
  --title "Logo update" --type logo --deadline "next friday"

# View client details with all deliverables
nex-deliverables client show --name "Ribbens Airco"

nex-deliverables list --status in_progress

# Update status
nex-deliverables status 1 in_progress
nex-deliverables mark "Homepage redesign" delivered

# Check workload
nex-deliverables workload

# Generate email
nex-deliverables email "Ribbens Airco"

# Export
nex-deliverables export --format csv --output report.csv

# Statistics
nex-deliverables stats
